Atributos de calidad

Son caracteristicas que queremos que cumpla
nuestro Software a la hora de progamarlo.

Caracteristicas

Atributo de
disponibilidad

La disponibilidad es la forma de medir en un periodo de tiempo que un servicio está disponible, y también cuando no está disponible.

Tacticas de disponibilidad

Algunas tacticas de disponibilidad

Deteccion de fallas

Estrategias para la deteccion de fallas que afecten nuestra disponibilidad

Recuperarse de fallas

Tactica para la recuperacion eficiente, en caso de sufrir alguna falla ya sea de hardware o software.

Prevenir fallas

Sistemas de prevencion de fallas, usar ciertos estandares de calidad en nuestros servidores para asi prevenir el mayor numero de fallas posible.

Atributo de
escalabilidad

La escalabilidad mide por la cantidad de solicitudes que una aplicación soporte, la escalabilidad define la facilidad que tiene una aplicación para escalar, ya sea de manera vertical u horizontal.

Tacticas de escalabilidad

Algunas tacticas de escalabilidad..

Escalamiento horizontal

Es la capacidad que tiene mi sistema para crecer en cuanto a hardware, ya sea aumento de disco duro, o memoria ram.

Escalamiento vertical

Es la capacidad que tiene mi sistema, para crecer en cuanto a software, nuevas funcionalidades, duplicar nodos o servicios.

Atributo de
interoperabilidad

La interoperabilidad,es la capacidad que tiene mi sistema de intercambiar datos entre si, de manera fácil y práctica.

Tacticas de interoperabilidad

Algunas tacticas de interoperabilidad...

Localizar

la localizacion, es la falicidad que tendra mi sistema para encontrar la informacion proveniente de diferentes fuentes de datos.

Gestionar interfaces

Es la facilidad de gestion que tengan mis interfaces, en mi sistema para intercambiar informacion de un lugar a otro.

Atributo de
modificabilidad

La modificabilidad, es la característica de mi software en donde se me facilita agregar cambios y agregar funcionalidades de manera fácil.

Tacticas de modificabilidad

Algunas tacticas de modificabilidad...

Reducir el tamaño de un modulo

Aqui se aplican buenas practicas a la hora de programar, para que lo que que estemos construyendo se haga de manera optima y en pocas lineas.

Aumentar la cohesion

El aumento de cohesion indica, la fiabilidad y calidad del modulo que desarrollo, asi mismo como su reutilizacion.

Reducir el acoplamiento

la reduccion del acoplamiento hace referencia a disminuir la interdependencia que tiene un modulo de otro modulo.

Atributo de
rendimiento

Es el desempeño relativo y la cantidad de recursos usados en determinado tiempo,la cantidad de operaciones que pueda realizar en determinado tiempo y uso de recursos.

Tacticas de rendimiento

Algunas tacticas de rendimiento...

Control de demanda de recursos

Es la fiabilidad que tiene mi sistema, para la administracion de cuanto puede consumir un recurso un especifico.

Administrar recursos

Nos indica facilidad con la que mi sistema administra sus recursos para asi ofrecer el maximo desempeño.

Atributo de
seguridad

El atributo de seguridad me indica como un sistema es capaz de proteger datos y evitar accesos no deseados o autorizados.

Tacticas de seguridad

Algunas tacticas de seguridad

Detectar ataques

la deteccion de ataques, es la facilidad con que mi sistema detecta que esta siendo vulnerado.

Resistir ataques

La capcidad que tiene mi sistema, para resistir y analizar los ataques provenientes de terceros.

Reaccionar a ataques

La reactividad a ataques, me indica como mi sistema reacciona y prevee ataques mal intencionados.

Atributo de
usabilidad

El atributo de usabilidad me indica, la facilidad con la que la mi sistema interactúa con el usuario.

Tacticas de usabilidad

Algunas tacticas de usabilidad...

Sistema de soporte

El sistema de soporte me indica, la capacidad de mi sistema para brindar soporte al usuario para facilitar el uso de la aplicacion.

Sistema de apoyo

El sistema de apoyo me indica, la capidad de mi sistema para apoyar el usario mientras realiza una tarea determinada en mi sistema.

Atributo de
desplegabilidad

La desplegabilidad me indica la facilidad con que mi sistema sube algun cambio a produccion.

Tacticas de desplegabilidad

Algunas tacticas de desplegabilidad..

Integracion continua

La integracion continua es una practica que me permite hacer integraciones automaticas en mi sistema.
